PDFs and Screen Readers

Note that the files below were crafted to include accessibility errors when read using a screen reader.
Can you identify the problems with each file? Do you know how to fix the problems?

Demo 1

March Break Activity Poster

Prerequisites

  • Adobe Acrobat Reader

Instructions

  • Open the PDF document in Adobe Acrobat Reader.
  • Use Acrobat's Read Out Loud feature to read the document.
Accessibility Problems (Click to Reveal)
  • No alternative text for the poster image.
    Alternative text is used to describe the content of an image. When no alternative text is available, nothing is communicated to screen reader users.

Demo 2

Children's Programming Newsletter

Prerequisites

  • Adobe Acrobat Reader

Instructions

  • Open the PDF document in Adobe Acrobat Reader.
  • Use Acrobat's Read Out Loud feature to read the document.
Accessibility Problems (Click to Reveal)
  • Screen reader reading order is not correct.
    Desktop publishing tools, like Microsoft Publisher, order content in the order it is added to the document. That doesn't always correspond to how the content should be read in the end. Use Adobe Acrobat Pro or the free PAC (PDF Accessibility Checker) tool to ensure the content is properly ordered.
  • No alternative text for informative images.
    Alternative text is used to describe the content of an image. When no alternative text is available, nothing is communicated to screen reader users. While some images are purely decorative and do not require alternative text descriptions, sponsor logo images should have alternative text so that the sponsor names are read.

Demo 3

Cemetery Burial Charges

Prerequisites

  • Adobe Acrobat Reader

Instructions

  • Open the PDF document in Adobe Acrobat Reader.
  • Use Acrobat's Read Out Loud feature to read the document.
Accessibility Problems (Click to Reveal)
  • Scanned text not properly recognized.
    Although it is possible to create PDF documents using office photocopiers, it should be avoided. The content made available to screen readers, if anything, is often littered with typos, and structural information, like table structures, are lost. Whenever possible, PDF documents should be created from the application that created the document.
